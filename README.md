# Clan Capital Base Bot

A Discord bot for managing Clash of Clans Clan Capital base layouts, clans, birthdays, and timezones. Hosted on **Render**, database on **Supabase (PostgreSQL)**.

Includes a local script (`renew_bases.py`) that opens every saved base link inside an Android emulator via ADB to keep layouts active.

---

## Stack

| Part | Technology |
|---|---|
| Bot hosting | [Render](https://render.com) (free web service) |
| Database | [Supabase](https://supabase.com) (PostgreSQL) |
| Language | Python 3.11+ |
| Discord library | discord.py |
| Emulator (local) | LDPlayer |

## Discord Slash Commands

### Bases
| Command | Description |
|---|---|
| `/add_base` | Add a CoC capital base layout link with optional screenshot and builder name |
| `/view_bases` | Browse bases with filters (district, builder, description search) |
| `/edit_base` | Edit an existing base by ID or link |
| `/remove_base` | Remove a base by its layout link |
| `/import_channel` | Bulk import bases from a Discord channel (scans messages for CoC links + screenshots) |

### Clans
| Command | Description |
|---|---|
| `/add_clan` | Register a clan with name, link, owner, and description |
| `/view_clans` | Browse registered clans with filters |
| `/edit_clan` | Edit an existing clan by ID or link |

### Timezones
| Command | Description |
|---|---|
| `/add_timezone` | Add a UTC offset (e.g. `+05:30`) with an optional label |
| `/remove_timezone` | Remove a timezone by offset |
| `/view_timezones` | Show all saved timezones with current local times |

### Birthdays
| Command | Description |
|---|---|
| `/set_birthday` | Set your birthday (month, day, timezone) |
| `/remove_birthday` | Remove your birthday |
| `/view_birthdays` | View all saved birthdays |

The bot automatically posts a birthday message at midnight in each user's timezone.

## Environment Variables

Set these in Render's environment settings (for the bot) and in `.env` locally (for `renew_bases.py`).

| Variable | Required | Description |
|---|---|---|
| `DISCORD_BOT_TOKEN` | ✅ | Your Discord bot token |
| `DISCORD_GUILD_ID` | ✅ | Your Discord server ID |
| `DATABASE_URL` | ✅ | Supabase PostgreSQL connection string |
| `BIRTHDAY_CHANNEL_ID` | ✅ | Discord channel ID for birthday announcements |
| `RENDER_URL` | optional | Your Render app URL — enables self-ping keepalive |
| `PORT` | optional | HTTP server port (default: 8080) |

### Getting the Supabase connection string

1. Open your Supabase project
2. Go to **Project Settings → Database → Connection string**
3. Select **URI** and copy it
4. Use the **Transaction pooler** (port 6543) for better connection handling

### Example `.env` for local use

```
DISCORD_BOT_TOKEN=your_bot_token_here
DISCORD_GUILD_ID=your_guild_id_here
DATABASE_URL=postgresql://postgres.<ref>:<password>@<host>:6543/postgres
BIRTHDAY_CHANNEL_ID=your_channel_id_here
RENDER_URL=https://your-app.onrender.com
```

## Database

Hosted on **Supabase (PostgreSQL)**. Tables are created automatically on bot startup via `database.py → init_db()` — no manual setup needed.

### `bases`
Stores Clan Capital base layout links.

| Column | Type | Description |
|---|---|---|
| `id` | SERIAL PK | Auto-incremented base ID |
| `district_number` | INTEGER | 0–8 (see district map below) |
| `link` | TEXT | CoC layout link |
| `screenshot` | TEXT | Screenshot image URL (optional) |
| `builder_name` | TEXT | Builder name (optional) |
| `description` | TEXT | Strategy description (optional) |
| `added_at` | TIMESTAMPTZ | Timestamp when added |

### `clans`
Stores registered clans.

| Column | Type | Description |
|---|---|---|
| `id` | SERIAL PK | Auto-incremented clan ID |
| `clan_name` | TEXT | Name of the clan |
| `clan_link` | TEXT UNIQUE | CoC clan link |
| `owner` | TEXT | Leader/owner name (optional) |
| `description` | TEXT | Clan description (optional) |
| `added_at` | TIMESTAMPTZ | Timestamp when added |

### `timezones`
Stores timezone offsets for the clock display.

| Column | Type | Description |
|---|---|---|
| `id` | SERIAL PK | Auto-incremented ID |
| `offset_str` | TEXT UNIQUE | UTC offset string e.g. `+05:30` |
| `label` | TEXT | Friendly name e.g. `India` (optional) |
| `added_at` | TIMESTAMPTZ | Timestamp when added |

### `birthdays`
Stores member birthdays for automatic announcements.

| Column | Type | Description |
|---|---|---|
| `id` | SERIAL PK | Auto-incremented ID |
| `discord_user_id` | TEXT UNIQUE | Discord user ID (snowflake as string) |
| `birth_month` | INTEGER | Birth month (1–12) |
| `birth_day` | INTEGER | Birth day (1–31) |
| `timezone_offset` | TEXT | UTC offset for midnight detection (default `+05:30`) |
| `added_at` | TIMESTAMPTZ | Timestamp when added |

## Deploying to Render

1. Push your code to GitHub
2. Create a new **Web Service** on Render, connect your repo
3. Set **Build Command**: `pip install -r requirements.txt`
4. Set **Start Command**: `python main.py`
5. Add all environment variables under **Environment**
6. Deploy

The bot includes a self-ping keepalive (`main.py → self_ping()`) to prevent Render's free tier from spinning down. Set `RENDER_URL` to your Render app URL to enable it.

## Local Setup (for renew_bases.py)

The renew script runs **on your local machine** with an Android emulator. It is completely separate from the bot hosted on Render.

### One-time setup

```cmd
pip install psycopg2-binary opencv-python numpy python-dotenv
```

Create a `.env` file in the project root:

```
DATABASE_URL=postgresql://postgres.<ref>:<password>@<host>:6543/postgres
```

### Running the renew script

Make sure:
- LDPlayer is running
- Clash of Clans is open and logged in
- Your village screen is visible (no popups open)

```cmd
python renew_bases.py                   # renew all bases in the database
python renew_bases.py --district 3      # renew Balloon Lagoon only
python renew_bases.py --debug           # verbose output with pixel check details
```

### What it does per base

For each base link the script:
1. Sends the layout link to CoC via ADB intent
2. Waits 5 seconds for the popup to appear
3. Checks pixel colors to confirm the base viewer is open
4. Taps through 5 buttons (slot select → OK → Cancel → OK → close X)
5. Moves to the next base

If a base fails all 3 attempts, it force-stops CoC, waits 20 seconds, relaunches the game, and continues with the next base. Failed bases are written to `failed_bases.log`.

### Recalibrating button coordinates

If taps stop registering correctly (after a game update or resolution change), run the calibration tool:

```cmd
python calibrate_pixels.py
```

It walks you through all 5 screens step by step, opens a pixel finder window for each, and prints a ready-to-paste `TAP_SEQUENCE` block at the end.
